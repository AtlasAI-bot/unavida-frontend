# Mastering Pharmacology — Chapter Production Blueprint

Use this as the standard checklist for every chapter so content quality stays consistent and launch-ready.

## 1) Chapter Identity
- Chapter number + title
- Course mapping (NUR1100 / NUR2110)
- Estimated completion time (minutes)
- Difficulty level (Intro / Intermediate / Advanced)

## 2) Learning Design Targets
- Total sections per chapter: **8–12 target**
- Learning objectives: **5–8 measurable objectives**
- Key takeaways: **5–10 concise bullets**
- Clinical outcomes: **3–5 applied competencies**

## 3) Core Content Requirements (Per Section)
Each section should include:
1. Section intro (why it matters clinically)
2. Main teaching content
3. Clinical example/scenario
4. Safety callout(s)
5. Section recap

Per chapter content minimums:
- Narrative instructional content: **6,000–10,000 words**
- Clinical scenarios embedded in reading: **3+**
- Safety-focused callouts (Exam Alert / Safety Alert): **6+**

## 4) Media Requirements
### Video
- Chapter-level videos: **3–8** (based on complexity)
- Per major concept: at least **1 short explainer**
- Format: section-mapped for Video Library + Reader relevance

### Images/Diagrams
- Diagrams/illustrations: **8–20 per chapter**
- Mandatory image coverage:
  - 1 concept map
  - 1 process/flow diagram
  - 1 safety-focused visual
  - 1 clinical decision tree (if applicable)

### Accessibility
- Alt text for all images
- Captions/transcripts for videos
- High contrast validation for callout boxes

## 5) Interactive Learning Assets
### Quizzes
- Question bank per chapter: **20–40 questions**
- Mix: recall + application + clinical judgment
- Rationales required for all items

### Flashcards
- Deck size per chapter: **30–60 cards**
- Include definitions, mechanisms, high-yield safety points

### Case Studies
- Case studies per chapter: **3–6**
- Difficulty distribution: beginner/intermediate/advanced
- Include linked section reference points

## 6) Reader/UX Integration Requirements
- TOC structure complete and sequenced
- Section labels standardized (`secX_Y_name`)
- Breadcrumb/navigation validated
- Exam Alert styling validated in light + dark mode
- Section formatting QA (lists, headings, spacing)

## 7) Assessment & Tracking Requirements
- Chapter completion logic validated
- Quiz scoring + pass threshold configured
- Flashcard progress tracking enabled
- Workbook compatibility (bookmarks/highlights)

## 8) QA Checklist (Definition of Done)
A chapter is “complete” only when:
- [ ] All sections authored + proofed
- [ ] Videos linked and playable
- [ ] Images loaded and quality-checked
- [ ] Quiz bank tested end-to-end
- [ ] Flashcards reviewed for clarity
- [ ] Case studies reviewed for realism/safety
- [ ] Reader formatting issues resolved
- [ ] Mobile + desktop pass completed
- [ ] Faculty/owner review approved
